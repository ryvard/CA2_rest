Test strategy

Vi har testet vores PersonFacade klasse med JUnit test. De fleste af vores tests virker
med undtagelse af vores addPerson test som fejler da vores addPerson metode i facaden ikke virker.
addPerson metoden virker ikke fordi der skal ligge noget i InfoEntity før vi kan ligge noget ned i
personen men det har vi desværre ikke nået at få sat op.
Vi har brugt browser og postman til at teste vores Rest API manuelt.
Vi har forsøgt at lave REST Assured for at teste vores REST API automatisk.

Inheritance strategy

Vi har brugt joined, da denne strategi laver den mest normaliserede database.
Dermed slipper vi blandtandet for mange redundante data.

Who did what
